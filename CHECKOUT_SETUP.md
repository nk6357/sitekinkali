# Система оформления заказов и отправки email

## Описание

Полная система оформления заказов с валидацией и отправкой писем администратору и клиенту.

## Функциональность

### Страница оформления заказа
- **Полная страница** (не модальное окно)
- Отображение товаров из корзины с ценами
- Форма контактных данных:
  - Имя (обязательно)
  - Номер телефона (обязательно, валидация формата)
  - Email (обязательно, валидация)
  - Комментарий (опционально, по умолчанию "Как можно скорее")
- Автозаполненные нередактируемые поля:
  - Место самовывоза: "Самовывоз Белинского, 6Б"
  - Способ оплаты: "Наличные при получении"
- Чекбокс согласия на обработку данных (по умолчанию не отмечен, обязателен)

### Email уведомления
**Администратору:**
- Детали заказа (товары, количество, сумма)
- Информация о клиенте (имя, телефон, email)
- Место самовывоза и комментарий
- Время получения заказа

**Клиенту:**
- Подтверждение заказа с галочкой ✓
- Полный список заказанных товаров
- Итоговая сумма
- Благодарность и обещание связаться

## Установка и конфигурация

### 1. Базовая настройка (уже сделана)

Проект содержит все необходимые пакеты:
```bash
npm install
```

### 2. Конфигурация Email (ОБЯЗАТЕЛЬНО)

Отредактируйте файл `.env` с вашей SMTP конфигурацией:

```env
# SMTP сервер (рекомендуется Mailtrap.io для тестирования)
SMTP_HOST=sandbox.smtp.mailtrap.io
SMTP_PORT=2525
SMTP_SECURE=false
SMTP_USER=ваш_пользователь
SMTP_PASSWORD=ваш_пароль
SMTP_FROM=noreply@sitekinkali.ru

# Email администратора для получения заказов
ADMIN_EMAIL=заказы@sitekinkali.ru

# Порт сервера
PORT=3000
```

### 3. Рекомендуемые email сервисы

#### Mailtrap.io (рекомендуется для тестирования)
1. Перейдите на https://mailtrap.io
2. Создайте бесплатный аккаунт
3. В панели управления перейдите в "Integrations" → "Nodemailer"
4. Скопируйте параметры SMTP в `.env`
5. Письма будут видны в веб-интерфейсе Mailtrap

#### Gmail
1. Включите двухфакторную аутентификацию
2. Создайте App Password: https://myaccount.google.com/apppasswords
3. Используйте этот пароль в SMTP_PASSWORD
4. Конфиг:
```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=ваша_почта@gmail.com
SMTP_PASSWORD=app_password_из_16_символов
SMTP_FROM=ваша_почта@gmail.com
ADMIN_EMAIL=admin@sitekinkali.ru
```

#### SendGrid (профессиональный вариант)
1. Зарегистрируйтесь на https://sendgrid.com
2. Создайте API ключ в настройках
3. Используйте в SMTP конфиге

## Архитектура

### Frontend
- **CheckoutPage** (`src/components/checkout/CheckoutPage.tsx`)
  - Форма оформления заказа
  - Валидация полей
  - Отправка запроса на `/api/orders/send-email`
  - Экран успеха

### Validation
- **src/utils/validation.ts**
  - `validateName()` - проверка имени
  - `validatePhone()` - формат телефона РФ
  - `validateEmail()` - формат email
  - `validateConsent()` - согласие обязательно

### Backend
- **server.js**
  - Express приложение
  - CORS включен
  - POST `/api/orders/send-email` - отправка писем
  - GET `/api/health` - проверка статуса

### State Management
- **CartContext** добавлены:
  - `isCheckout` - флаг открытия страницы
  - `openCheckout()` - открыть страницу
  - `closeCheckout()` - закрыть страницу

## Поток использования

1. Пользователь добавляет товары в корзину
2. Нажимает "Оформить предзаказ" в корзине
3. Открывается страница оформления заказа
4. Заполняет форму контактных данных
5. Отмечает чекбокс согласия
6. Нажимает "Оформить предзаказ"
7. Если валидация прошла:
   - На email администратора отправляется заказ
   - На email клиента отправляется подтверждение
   - Показывается экран успеха
   - Корзина очищается
8. Через 2 секунды возвращается на сайт

## Информация об email

### Формат писем
- HTML формат с профессиональным оформлением
- Русская локализация дат и сумм
- Таблица товаров с колонками: название, кол-во, сумма

### Тестирование
1. Используйте Mailtrap.io для локального тестирования
2. Все письма будут видны в веб-интерфейсе
3. Не нужен реальный email сервис на этапе разработки

## Переменные окружения

| Переменная | Описание | Пример |
|-----------|---------|--------|
| SMTP_HOST | SMTP сервер | sandbox.smtp.mailtrap.io |
| SMTP_PORT | Порт SMTP | 2525 |
| SMTP_SECURE | Использовать TLS | false |
| SMTP_USER | Пользователь SMTP | your_user |
| SMTP_PASSWORD | Пароль SMTP | your_password |
| SMTP_FROM | От кого отправлять письма | noreply@sitekinkali.ru |
| ADMIN_EMAIL | Email администратора | orders@sitekinkali.ru |
| PORT | Порт Express сервера | 3000 |

## Обработка ошибок

### Ошибки валидации
Показываются под каждым полем формы в красном цвете.

### Ошибки отправки
- Если письмо не отправилось, показывается сообщение об ошибке
- Проверьте:
  - Параметры SMTP в `.env`
  - Интернет соединение
  - Если используете Gmail - правильность app password

### Логи сервера
Все ошибки логируются в консоль Express сервера для отладки.

## Развертывание в production

### Переменные окружения
На сервере установите реальные параметры email сервиса в переменные окружения.

### Рекомендации
1. Используйте SendGrid или другой профессиональный сервис
2. Установите SSL/TLS для SMTP
3. Добавьте rate limiting для API
4. Настройте мониторинг отправки писем
5. Сохраняйте логи заказов в базе данных
