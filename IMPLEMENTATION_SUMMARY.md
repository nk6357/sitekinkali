# Реализация системы оформления заказов

## ✅ Что реализовано

### 1. Страница оформления заказа (CheckoutPage)
- **Полная страница** вместо модального окна
- Отображение товаров из корзины с ценами и количеством
- Форма с полями:
  - ✅ Имя (обязательное, мин. 2 символа)
  - ✅ Номер телефона (обязательное, валидация формата РФ)
  - ✅ Email (обязательное, валидация формата)
  - ✅ Комментарий (опциональное, placeholder с указанием времени)
  - ✅ Согласие на обработку данных (чекбокс, по умолчанию НЕ отмечен)
- Нередактируемые поля:
  - ✅ "Самовывоз Белинского, 6Б"
  - ✅ "Наличные при получении"
- Кнопка закрытия (X) в хедере
- Экран успеха с галочкой после отправки

### 2. Система валидации (validation.ts)
- Проверка имени (обязательно, мин. 2 символа)
- Валидация телефонного номера (10+ цифр, различные форматы)
- Валидация email (стандартный формат)
- Проверка согласия на обработку данных

### 3. Backend API (server.js)
- Express сервер на порту 3000
- POST `/api/orders/send-email`
- CORS включен для фронтенда
- Отправка HTML писем через Nodemailer

### 4. Email уведомления
**Администратору:**
- Новый заказ с информацией о клиенте
- Полный список товаров в таблице
- Сумма заказа
- Комментарий и время

**Клиенту:**
- Подтверждение ✓ Ваш заказ принят
- Список заказанных блюд
- Итоговая сумма
- Обещание связаться

### 5. Интеграция с корзиной
- Кнопка "Оформить предзаказ" в корзине открывает CheckoutPage
- После успешной отправки корзина очищается
- Через 2 сек. пользователь возвращается на сайт

### 6. Управление состоянием
- **CartContext** расширен:
  - `isCheckout: boolean` - флаг открытия страницы
  - `openCheckout()` - открыть страницу
  - `closeCheckout()` - закрыть страницу

## 🚀 Как использовать

### Установка зависимостей
```bash
npm install
```

### Конфигурация email (обязательно!)
Отредактируйте `.env`:
```env
SMTP_HOST=sandbox.smtp.mailtrap.io      # Пример Mailtrap
SMTP_PORT=2525
SMTP_SECURE=false
SMTP_USER=ваш_user
SMTP_PASSWORD=ваш_password
ADMIN_EMAIL=admin@example.com
```

Рекомендуется Mailtrap.io для тестирования:
1. https://mailtrap.io - создать бесплатный аккаунт
2. Скопировать SMTP параметры в `.env`
3. Письма будут видны в веб-интерфейсе

### Запуск разработки
```bash
npm run dev          # Запускает Vite (5173) + Express (3000)
```

### Тестирование
1. Откройте http://localhost:5173
2. Добавьте товары в корзину
3. Нажмите "Оформить предзаказ"
4. Заполните форму и отправьте
5. Проверьте Mailtrap.io или почту администратора

### Продакшн сборка
```bash
npm run build        # Сборка проекта
npm run preview      # Превью продакшн версии
```

## 📝 Примечания

### Валидация
- Все поля кроме комментария - обязательные
- Согласие на обработку данных по умолчанию НЕ отмечено
- Телефон поддерживает различные форматы РФ
- Email проверяется базовым regex

### Email
- Форматированные HTML письма
- Русская локализация дат и сумм (toLocaleString)
- Таблица товаров с цветным оформлением
- Отправка администратору И клиенту одновременно

### Поток заказа
1. Пользователь открывает CheckoutPage
2. Заполняет и валидирует форму
3. Отправляет запрос на `/api/orders/send-email`
4. Сервер отправляет 2 письма (администратор + клиент)
5. Показывается экран успеха
6. Корзина очищается
7. Возврат на главную страницу

3. Добавьте rate limiting
4. Логируйте заказы в БД
5. Настройте мониторинг
